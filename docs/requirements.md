# Glint - 要件定義書

## 1. プロダクト概要

### 1.1 コンセプト
**「既存のめんどくさい物件探しを脱却して、最短で、マッチングアプリみたいに物件探しができる」**

AIアドバイザーが、エリア→物件の2段階でマッチング形式で提案してくれる賃貸物件探しアプリ。

### 1.2 プロダクト名
**Glint**（グリント）

### 1.3 解決する課題

#### ユーザーの現状の不満（ペインポイント）

| カテゴリ | 不満・課題 |
|----------|-----------|
| 地域選び | 行政サポート・補助金を調べるのが面倒 |
| | 職場からの乗り換え・運賃・最短距離を調べるのが面倒 |
| | 治安情報を調べるのが面倒 |
| | 災害時の安全マップを確認するのが面倒 |
| 場所選び | 周辺にスーパーがあるか調べるのが面倒 |
| | 駅の数・路線を調べるのが面倒 |
| | コンビニ・病院・学校・公園の有無を調べるのが面倒 |
| | 交通量を調べるのが面倒 |
| 物件選び | 各サイトで掲載テンプレが違い、比較しづらい |
| | 情報が細かく羅列していて見づらい |
| | 欲しい情報が載っていないのか見つけられないのか分からない |
| | 写真だけでは間取り・きれいさ・条件が判断しづらい |

### 1.4 ターゲットユーザー
- **年齢層**: 20〜30代
- **世帯**: 単身者、カップル、ファミリー（すべて対応）
- **引っ越し経験**: 問わない（初めて〜経験者まで）
- **対象エリア**: 関東圏（東京都、神奈川県、埼玉県、千葉県）

## 2. 主要機能

### 2.1 コア機能一覧

| # | 機能名 | 概要 |
|---|--------|------|
| 1 | オンボーディング | 初回登録時のヒアリング（ライフスタイル・条件） |
| 2 | エリアスワイプ | エリアをスワイプで選択（第1段階） |
| 3 | 物件スワイプ | 物件をスワイプで選択（第2段階） |
| 4 | 相性%表示 | 各エリア・物件に対するマッチング度を%で表示 |
| 5 | 自動情報取得 | 通勤時間・治安・災害・周辺施設を自動取得・表示 |
| 6 | AI助言機能 | 「ここ大丈夫？」という懸念点の指摘 |
| 7 | トレードオフ提案 | 「条件を緩めればこんな選択肢も」という提案 |
| 8 | 学習・提案機能 | スワイプ傾向から隠れた好みを発見・提案 |
| 9 | お気に入り保存 | いいねした物件のリスト保存 |
| 10 | 外部サイト連携 | 物件詳細・問い合わせは外部サイトへリンク |
| 11 | 通知機能 | 好みに合う新着物件の通知 |

### 2.2 機能詳細

#### 2.2.1 オンボーディング（初回ヒアリング）

結婚相談所の初回カウンセリングを参考に設計。

**Step 1: 基本情報**
| 項目 | 入力形式 | 必須 |
|------|---------|------|
| 職場の住所 | 住所入力（複数可） | ✅ |
| 引っ越し希望時期 | 選択式 | ✅ |
| 予算（家賃上限） | スライダー or 入力 | ✅ |
| 入居人数 | 選択式（1人/2人/3人以上） | ✅ |

**Step 2: ライフスタイル**
| 項目 | 入力形式 | 必須 |
|------|---------|------|
| 在宅勤務の頻度 | 選択式（毎日/週数回/なし） | ✅ |
| 帰宅時間帯 | 選択式（早め/普通/遅め/不規則） | ✅ |
| 休日の過ごし方 | 選択式（インドア/アウトドア/両方） | ✅ |
| 車/自転車の有無 | チェックボックス | ✅ |

**Step 3: 重視すること**
| 項目 | 入力形式 |
|------|---------|
| 優先度選択 | 治安/広さ/新しさ/駅近/静かさ/日当たり 等から優先順位付け |

**Step 4: 絶対NG条件**
| 項目 | 入力形式 |
|------|---------|
| NG条件選択 | 1階NG/ユニットバスNG/3点ユニットNG/プロパンガスNG 等 |

**Step 5: 過去の経験（任意）**
| 項目 | 入力形式 |
|------|---------|
| 前の家で不満だったこと | 自由入力 or 選択式 |
| 引っ越し経験の有無 | 選択式 |

#### 2.2.2 エリアスワイプ（第1段階）

- エリアカードをTinder風にスワイプ
- 各エリアに **相性%** を表示
- カード表示内容:
  - エリア名・写真
  - 相性%
  - 通勤時間（登録した職場から）
  - 家賃相場
  - 治安スコア
  - 主要駅・路線

#### 2.2.3 物件スワイプ（第2段階）

- 選んだエリア内の物件をスワイプ
- 各物件に **相性%** を表示
- カード表示内容:
  - 物件写真
  - 相性%
  - 家賃
  - 間取り
  - 駅徒歩
  - 築年数
  - 特徴タグ

#### 2.2.4 自動情報取得・表示

ユーザーが「調べるのが面倒」と感じていた情報を自動取得し、必要な項目だけ表示。

**地域情報**
| 情報 | データソース候補 |
|------|-----------------|
| 行政サポート・補助金 | 各自治体API/Webスクレイピング |
| 通勤時間・乗り換え・運賃 | Google Maps API / NAVITIME API |
| 治安情報 | 警視庁犯罪情報マップ等 |
| 災害ハザードマップ | 国土交通省 不動産情報ライブラリAPI |

**場所情報**
| 情報 | データソース候補 |
|------|-----------------|
| 周辺スーパー・コンビニ | Google Places API |
| 駅・路線情報 | 駅データ.jp / Google Maps API |
| 病院・学校・公園 | Google Places API |
| 交通量 | 国土交通省 道路交通センサス |

#### 2.2.5 AI助言機能

ユーザーが気づいていない懸念点を指摘する機能。

**トリガー例:**
- 「この物件、ハザードマップで浸水リスクがありますが大丈夫ですか？」
- 「このエリア、夜間の治安データが少し気になりますが確認しましたか？」
- 「築年数が古いですが、耐震基準は確認しましたか？」

**トーン:** あくまで助言・お伺い。押し付けない。

#### 2.2.6 トレードオフ提案

マッチングアプリ/結婚相談所的な「条件を緩めれば、こんな選択肢も」提案。

**提案例:**
- 「家賃を5,000円上げると、こんな良い物件も選べますよ」
- 「希望エリアの隣駅、実は治安が良くて家賃も安いですよ」
- 「駅徒歩15分だけど、自転車なら5分。駐輪場無料です」
- 「築年数は古いけど、リノベ済みでこの広さはお得ですよ」

#### 2.2.7 学習・提案機能

スワイプ傾向からユーザーの隠れた好みを発見し、提案する。

**フロー:**
1. ユーザーのスワイプパターンを分析
2. 共通する特徴を検出（例: 「角部屋」を無意識に選んでいる）
3. スワイプ中にポップアップで提案:
   - 「あなたが気づいてない、物件の好みを見つけちゃいました💡」
   - 「もしかして『角部屋』も重視してますか？」
4. 「はい」→ 「それでは条件に追加しちゃいましょう！」（事後報告風）
5. 以降のスワイプに条件として反映

**中間択:** 要検討（心理的負荷を考慮）

#### 2.2.8 通知機能

- 「あなたの好みそうな物件が増えましたよ！」
- 新着物件で相性%が高いものがあれば通知

## 3. ユーザーフロー

```
[アプリ起動]
    ↓
[オンボーディング（初回のみ）]
    ├─ Step1: 基本情報
    ├─ Step2: ライフスタイル
    ├─ Step3: 重視すること
    ├─ Step4: 絶対NG条件
    └─ Step5: 過去の経験（任意）
    ↓
[ホーム画面]
    ↓
[エリアスワイプ]
    ├─ 右スワイプ: いいね（エリア候補に追加）
    ├─ 左スワイプ: パス
    └─ AI助言/トレードオフ提案が差し込まれる
    ↓
[物件スワイプ]
    ├─ 選んだエリア内の物件をスワイプ
    ├─ 右スワイプ: いいね（お気に入りに追加）
    ├─ 左スワイプ: パス
    ├─ 学習機能: 「この条件も気にしてますか？」
    └─ AI助言/トレードオフ提案が差し込まれる
    ↓
[お気に入りリスト]
    ├─ いいねした物件一覧
    └─ 物件タップ → 外部サイトへリンク（SUUMO等）
```

## 4. 技術仕様

### 4.1 技術スタック

| レイヤー | 技術 |
|---------|------|
| モバイルアプリ | React Native（iOS/Android両対応） |
| バックエンド | 未定（Node.js / Python 等を想定） |
| データベース | 未定（PostgreSQL / Firebase 等を想定） |
| AI/ML | 未定（レコメンドエンジン、パターン分析） |

### 4.2 外部API連携

| API | 用途 | 費用 |
|-----|------|------|
| 国土交通省 不動産情報ライブラリAPI | 価格相場、ハザードマップ、周辺施設 | 無料 |
| Google Maps Platform | 通勤時間計算、周辺施設検索 | 従量課金 |
| Google Places API | スーパー、コンビニ、病院等の検索 | 従量課金 |
| 駅データ.jp 等 | 駅・路線情報 | 要確認 |

### 4.3 物件データについて

**制約:**
- SUUMO、LIFULL HOME'S等の大手は公開APIを提供していない
- レインズは宅建業者のみアクセス可能

**対応方針:**
1. **MVP段階**: ダミーデータ or 手動収集データで検証
2. **本格展開時**: 不動産会社との提携 or 不動産テック企業からのデータ購入を検討

## 8. 差別化ポイント

| 既存サービス | Glint |
|-------------|-------|
| 条件入力 → 一覧表示 → 比較 | スワイプで直感的に選択 |
